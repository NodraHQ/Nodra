# MODULE_TEMPLATE

Version: 1.0
Status: Stable

---

# Purpose

This document defines the standard structure for every Nodra Academy module.

All modules must follow this template.

Only educational content changes.

The overall structure remains identical.

---

# Module Information

Module Number

Module Name

Estimated Reading Time

Difficulty

Learning Objectives

---

# Hero Section

Module Title

Module Description

Estimated Reading Time

Difficulty Badge

Progress Indicator

---

# Learning Objectives

Every module begins with clear learning objectives.

Recommended amount:

3 to 5 objectives.

---

# Main Content

The lesson is divided into multiple sections.

Each section contains:

Title

Explanation

Examples

Illustrations (optional)

Notes (optional)

Warnings (optional)

---

# Section Structure

Each section should follow this order whenever possible.

Introduction

Explanation

Real-world Example

Important Note

Key Takeaway

---

# Educational Style

Prefer:

Simple language

Short paragraphs

Practical examples

Progressive difficulty

Real-world analogies

Avoid:

Academic language

Long blocks of text

Unexplained technical terms

Unnecessary repetition

---

# Callout Boxes

When appropriate, use:

Tip

Important

Warning

Did you know?

Best Practice

Only use existing Academy components.

Never create new callout styles.

---

# Images

Images are optional.

When used, they must reinforce learning.

Images must never replace explanations.

---

# Summary

Every module ends with a concise summary.

Recommended length:

5 to 10 bullet points.

---

# Glossary

Every module contains a glossary.

Only include concepts introduced during that module.

Avoid repeating definitions from previous modules.

---

# Quiz

Every module contains one quiz.

Recommended:

8 to 10 questions.

Questions should progressively increase in difficulty.

Prefer conceptual understanding over memorization.

---

# Quiz Structure

Question

Options

Correct Answer

Explanation

Learning Feedback

Reuse the existing quiz system.

---

# Navigation

Every module contains:

Previous Module

Next Module

Back to Academy

Maintain identical navigation behavior.

---

# Translation

Every visible text must use translation keys.

Never hardcode translatable text.

Each module uses its own namespace.

---

# Accessibility

Maintain heading hierarchy.

Preserve ARIA attributes.

Keep keyboard navigation working.

Maintain sufficient contrast.

---

# Responsive Behavior

Every section must remain responsive.

Desktop

Notebook

Tablet

Mobile

No additional responsive systems should be created.

---

# Module Completion Checklist

Before considering the implementation complete, verify:

✓ Hero completed

✓ Learning objectives

✓ All lesson sections

✓ Summary

✓ Glossary

✓ Quiz

✓ Navigation

✓ Translation keys

✓ Responsive layout

✓ No console errors

---

# Final Principle

Every module should look, feel and behave as if it was created on the same day by the same team.

Consistency is the highest priority.
